# 퇴근력 이벤트 스키마 (MVP)

## 이벤트 설계 원칙
- 모든 이벤트는 `user_id`, `session_id`, `platform`, `ts` 공통 필드를 포함한다.
- 광고 이벤트와 제품 이벤트를 같은 세션으로 연결한다.
- 퍼널 누락 점검을 위해 핵심 이벤트는 반드시 쌍으로 기록한다. (view -> complete)

## 공통 파라미터
- `user_id`: 익명 해시 ID
- `session_id`: 앱 진입 세션 ID
- `day_bucket`: YYYY-MM-DD (KST)
- `time_slot`: morning | evening | other
- `app_version`: 배포 버전

## 제품 이벤트
### 1) 온보딩
- `onboarding_started`
- `goal_selected`
  - params: `goal_type` (work|health|study|relationship)
- `reminder_time_set`
  - params: `morning_time`, `evening_time`
- `onboarding_completed`

### 2) 아침 체크인
- `morning_checkin_viewed`
- `morning_task_selected`
  - params: `task_source` (preset|custom), `goal_type`
- `morning_checkin_completed`
  - params: `input_length_bucket` (short|medium|long)

### 3) 저녁 체크아웃
- `evening_checkout_viewed`
- `evening_result_selected`
  - params: `result` (done|not_done)
- `evening_failure_reason_selected`
  - params: `reason` (time|tired|priority|difficulty)
- `evening_checkout_completed`
  - params: `result`

### 4) 성취/리포트
- `daily_result_viewed`
  - params: `score`, `streak_days`, `weekly_rate`
- `weekly_report_viewed`
  - params: `weekly_rate`, `wow_delta`
- `badge_earned`
  - params: `badge_id`, `badge_level`

## 광고 이벤트
- `ad_slot_requested`
  - params: `slot` (daily_result_gate|weekly_report_gate|home_banner)
- `ad_loaded`
  - params: `slot`, `ad_type` (interstitial|banner)
- `ad_failed`
  - params: `slot`, `error_code`
- `ad_shown`
  - params: `slot`, `ad_type`
- `ad_dismissed`
  - params: `slot`, `watch_seconds`
- `ad_click`
  - params: `slot`

## KPI 계산식
- D1/D7: `onboarding_completed` 코호트 기준 재방문
- 평균 일세션: `distinct(session_id) / distinct(user_id)` per day
- 저녁 완료율: `evening_checkout_completed / evening_checkout_viewed`
- 광고 이탈율: `ad_shown 후 앱 종료 / ad_shown`
- 광고 유효노출: `ad_dismissed / DAU`

## 대시보드 최소 뷰
- 퍼널: onboarding -> morning complete -> evening complete -> result view
- 리텐션: D1, D3, D7, D14
- 광고: slot별 fill rate, shown rate, exit rate, eCPM
- 품질: ad_failed rate, 이벤트 누락률
