# 퇴근력 new_project5 실행계획 (Git + Vercel)

## 기준 저장소/배포
- GitHub Repo: `leechang1811-afk/new_project5`
- Vercel Project: [new-project5](https://vercel.com/leechang1811-afks-projects/new-project5)

## 1) 즉시 해야 할 일 (우선순위)
1. Git 연결 상태 확인 (Vercel에서 Production 브랜치 매핑)
2. MVP 화면 5개 구현
3. 이벤트 스키마 삽입
4. 광고 슬롯 3개 삽입 + fallback 처리
5. 프로덕션 배포 후 핵심 퍼널 검증

## 2) 개발 작업 묶음 (작게 배포)
### Batch A (제품 뼈대)
- 온보딩 / 아침 체크인 / 저녁 체크아웃
- 결과 카드 UI
- 로컬 상태 저장

### Batch B (성취/리포트)
- streak, 주간 실행률, 전주 대비 계산
- 주간 리포트 화면
- 배지 기본 3종

### Batch C (광고/측정)
- banner + interstitial 슬롯 연결
- 이벤트 로깅 연결
- 광고 실패 fallback

## 3) 검증 시나리오
- 신규 유저 온보딩 완료 가능 여부
- 아침 체크인 30초 내 완료 여부
- 저녁 체크아웃 완료 후 결과 진입 정상 여부
- 광고 미로딩 시 결과 자동 진입 여부
- 주간 리포트 진입/이탈 추적 여부

## 4) 배포 체크리스트
- [ ] lint 통과
- [ ] build 통과
- [ ] 프로덕션 배포 성공
- [ ] 실기기(WebView)에서 광고/레이아웃 확인
- [ ] 이벤트 대시보드 수집 확인

## 5) 최종 리뷰 때 확인할 항목
- 제품 적합성: "30대가 하루 2번 쓸 이유"가 명확한가
- 수익성: 광고 노출이 리텐션을 해치지 않는가
- 안정성: 광고 실패/네트워크 불안정에서도 핵심 플로우가 유지되는가
- 확장성: 2주 실험으로 개선 방향이 보이는가
