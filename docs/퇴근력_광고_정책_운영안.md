# 퇴근력 광고 정책/운영안 (앱인토스 가정)

## 1) 광고 운영 목표
- 리텐션을 해치지 않으면서 ARPDAU를 늘린다.
- 사용자당 광고 피로를 통제한다.

## 2) 기본 정책
- Banner: 홈 하단 고정 1개
- Interstitial: 저녁 결과 진입 전 1회 중심
- Weekly Report Gate: 주간 리포트 진입 전 1회 (빈도 제한)
- Rewarded: MVP에서는 미적용 (2차 실험에서 도입)

## 3) 빈도 캡 (중요)
- Interstitial 일일 상한: 2회
- 같은 슬롯 연속 노출 금지: 최소 4시간 쿨다운
- 신규 유저 첫날: 전면 1회로 제한
- D1 이전 유저: 배너 우선, 전면 보수적으로 노출

## 4) 슬롯 정의
- `home_banner`: 홈/대시보드 하단
- `daily_result_gate`: 저녁 체크아웃 완료 후 결과 진입 전
- `weekly_report_gate`: 주간 리포트 진입 전

## 5) 실패 대응
- 광고 load 실패 시 1.5초 내 콘텐츠 자동 진입
- 광고 미노출이어도 보상/결과는 동일 제공
- 슬롯별 실패율 10% 초과 시 즉시 점검

## 6) UX 보호 가이드
- 광고 노출 전에 짧은 안내 문구 제공
  - 예: "잠시 후 오늘 결과를 보여드려요."
- CTA와 배너 겹침 금지 (공통 bottom offset)
- 사용자가 막혔다고 느끼는 정지 상태 금지

## 7) 수익/리텐션 동시 관리 지표
- `광고`: fill rate, show rate, eCPM, ad exit rate
- `제품`: evening completion rate, D1/D7, avg sessions
- 운영 룰:
  - ad exit rate > 10%면 즉시 빈도 하향
  - evening completion이 3일 연속 하락하면 결과 게이트 위치 재검토

## 8) 2주 실험 플랜
- 실험 A: 저녁 결과 게이트(현행)
- 실험 B: 주간 리포트 게이트 중심 + 저녁 게이트 완화
- 판단 기준:
  - D7 +2%p 이상 개선이면 B 유지
  - ARPDAU 감소폭이 8% 이하면 B 채택 가능
