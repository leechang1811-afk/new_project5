# 퇴근력 MVP PRD (new_project5)

## 1) 제품 한 줄 정의
- 30대 직장인이 하루 2번(출근 전/퇴근 후) 30~60초로 실행률을 올리고, 성취를 수치로 확인하는 초단기 루틴 앱.

## 2) 문제 정의
- 계획은 세우지만 실행이 끊긴다.
- 할 일 앱은 많지만, "매일 완료하게 만드는 최소 행동 엔진"은 부족하다.
- 광고 수익 모델에서 중요한 짧은 재방문 루프가 약하다.

## 3) 타깃
- Primary: 30~39세 직장인
- Secondary: 25~34세
- JTBD: "일이 많아도 하루 1개는 끝냈다는 느낌을 얻고 싶다."

## 4) 목표 지표 (런칭 후 4주)
- D1 Retention: 45%+
- D7 Retention: 25%+
- Avg Sessions / DAU: 1.8+
- Evening Check-out Completion Rate: 55%+
- Interstitial Exit Rate: 8% 이하

## 5) 핵심 가치 제안
- 아침: 오늘 실행할 1개를 작게 정한다.
- 저녁: 완료 여부를 체크하고 방해 요인을 기록한다.
- 즉시: 실행률, 연속일, 개선률을 보여준다.

## 6) MVP 범위 (Must)
- 온보딩: 목표 카테고리 1개 선택 (업무/건강/학습/관계)
- 아침 체크인: "오늘의 1개" 선택/입력
- 저녁 체크아웃: 완료/미완료 + 이유 1탭
- 성취 대시보드: 연속일, 주간 실행률, 전주 대비 개선률
- 리마인드: 하루 2회 알림 설정값 저장

## 7) 제외 범위 (Later)
- 소셜 피드/친구 경쟁
- 복잡한 캘린더/프로젝트 관리
- 장문 회고 에디터
- AI 대화형 코칭

## 8) 사용자 플로우
1. 첫 실행 -> 목표 선택 -> 알림 시간 설정
2. 아침 알림 -> 체크인 30초 완료
3. 저녁 알림 -> 체크아웃 60초 완료
4. 결과 카드 확인 -> 리포트/배지 확인
5. 다음날 자동 이어서 반복

## 9) 기능 요구사항
- 체크인 입력은 1~2개 액션으로 종료되어야 함
- 체크아웃은 완료/미완료 모두 2탭 이내로 종료
- 실패 입력 시 죄책감 카피 금지, 복귀 유도 문구 제공
- 광고 실패 시 핵심 기능은 항상 계속 동작

## 10) 논기능 요구사항
- 모바일 WebView 우선 레이아웃
- 하단 배너와 CTA 겹침 방지 (단일 offset source)
- 이벤트 로깅 누락률 1% 이하
- 페이지 전환 1초 이내 체감

## 11) 리스크와 대응
- 리스크: "또 할 일 앱" 인식
  - 대응: 완료 개수보다 실행률/연속일 중심 브랜딩
- 리스크: 광고 과다로 이탈
  - 대응: 전면 하루 1~2회 캡, 저녁 결과 직전 중심
- 리스크: 1주 후 흥미 하락
  - 대응: 7일/14일 챌린지, 주간 성장 피드백 카드

## 12) 출시 판정 기준
- Continue: D7 25%+, 저녁 완료율 55%+, 전면광고 이탈 8% 이하
- Iterate: D7 18~25%, 완료율 40~55%
- Stop/Pivot: D7 18% 미만 + 평균 일세션 1.3 미만
